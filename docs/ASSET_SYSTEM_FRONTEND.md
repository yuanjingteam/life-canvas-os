# 资产系统（ASSET）前端技术文档（草案）

> 目标：在 Life Canvas OS 中新增“资产系统（ASSET 维度）”模块的前端信息架构与页面/功能规划，便于后续按 MVP 落地实现。

## 0. 设计风格与一致性要求

- **视觉风格**：与现有模块（如 `FuelSystemPage`）保持一致，采用玻璃拟态风格与分区卡片布局。
  - 容器统一使用 `GlassCard`
  - 页面内以卡片分区组织信息层级（无 Tabs）
- **交互与状态**：加载态/空态/错误态表现与现有页面一致（如统一骨架/占位文案、`PageLoading`）。
- **组件栈一致**：优先使用 shadcn/ui（`Button/Input/Select/Popover/Badge` 等）与现有工具组件。
- **“实时”定义（前端）**：用户新增/编辑/删除资产后**立即刷新总资产与分类构成**；同时支持页面聚焦自动刷新（可选轮询）。


## 1. 入口与路由接入

### 1.1 侧边栏导航（NAV_ITEMS）
- 在 `src/renderer/lib/constants.tsx` 的 `NAV_ITEMS` 中新增：
  - `id: 'asset'`
  - `label: '资产系统'`
  - `icon`: 建议使用 `lucide-react` 的 `Wallet / Landmark / PiggyBank`

### 1.2 Sidebar 路由映射（routeMap）
- 在 `src/renderer/components/layout/Sidebar.tsx` 的 `routeMap` 新增：
  - `asset: '/system/asset'`

### 1.3 Router 新增路由
- 在 `src/renderer/routes.tsx` 新增懒加载页面：
  - `AssetSystemPage`
- 路径建议：
  - `/system/asset`

> 页面组织建议：使用“一个系统页 + 卡片分区”承载核心内容，主页面聚焦总览 + 分类卡片 + 新增资产表单。

## 2. 页面清单与功能规划

### 2.1 资产系统主页面：`AssetSystemPage`
**定位**：实时掌握总资产与结构组成；在同一页面完成新增资产与分类概览。

#### 页面结构（无 Tabs）
- 顶部标题区：标题 + 简短提示语
- 主区两列布局（保持同排）：
  - **左侧列**：
    - **总资产大卡**：展示总资产；卡片内包含“总负债 / 净资产”两个胶囊子卡
    - **净资产趋势卡**：展示趋势占位/折线
  - **右侧列**：
    - **新增资产常驻卡片**：直接内嵌表单（非入口）
      - 字段：资产名称、分类（搜索已有分类）、金额、保存操作
      - 分类输入为“可搜索的选择器”风格：无匹配时提示“新建分类 XXX”
      - 不提供独立“新增资产”按钮入口

#### 资产分类卡片区
- 分类以卡片网格展示（每类一个卡片）：
  - 展示：分类名、金额、占比、条目数、emoji/轻渐变
  - 点击卡片进入分类二级页：`/system/asset/categories/:category`
- 新增分类：
  - 末尾固定「+」卡片
  - 点击后直接插入“新分类X”卡片并自动聚焦输入

#### 资产项列表
- 不在主页面展示
- 仅在分类二级页展示该分类下的资产项列表、合计、编辑/删除

### 默认资产类别（建议预置，可自定义增删）

> 预置的目的：降低首次使用门槛；用户仍可自由新增/改名/调整归类。

默认预置（当前版本全部保留）：
- **现金**：现金、零钱
- **活期/储蓄**：银行卡活期、储蓄账户
- **投资理财**：基金、股票、债券、黄金、加密资产
- **固定资产**：房产、车辆、贵重物品、设备
- **应收款**：他人欠款、借出款
- **负债**：信用卡欠款、花呗/白条、贷款（房贷/车贷/消费贷）
- **保险/公积金/养老金**：社保/公积金、商业保险现金价值
- **其他**：无法归类的资产项（兜底分类）

命名建议：
- 类别名尽量使用“名词短语”，避免动词（例如用“活期/储蓄”而不是“存款”）
- 若未来不做多币种，类别不需要带币种信息


## 3. 关键交互与组件建议

### 3.1 新增资产卡片（常驻表单）
- 表单字段：资产名称、分类（搜索已有分类）、金额
- 分类选择器交互：
  - 输入即搜索已有分类
  - 无匹配时提示“新建分类 XXX”
- 保存后联动刷新总览与分类卡片统计

### 3.2 分类卡片交互
- 点击分类卡片进入二级页（分类详情）
- 新增分类：通过「+」卡片插入并自动聚焦输入
- 允许直接编辑分类名（Input 失焦/回车保存）

### 3.3 分类详情页（Category Detail Page）
- 展示该分类的资产项列表与合计
- 支持新增/编辑/删除资产项
- 支持空态引导（提示新增资产）

## 4. 与现有模块的集成点

## 4.1 “实时总资产/分类统计”的前端实现建议

- 数据来源建议拆为两类查询：
  - 汇总：总资产/总负债/净资产、分类构成（用于总览与分类卡片）
  - 明细：分类详情列表、资产项列表（用于二级页管理）
- 刷新策略（满足“实时”体验）：
  - 新增/编辑/删除资产项成功后：`invalidateQueries` 刷新汇总与相关明细
  - 页面聚焦时：触发 `refetch`（可选）
  - 需要更“实时”时：对汇总 query 增加 `refetchInterval`（例如 10~30s，可配置）
- 下钻交互：点击分类卡片进入 `/system/asset/categories/:category` 展示该类别资产项。


## 5. 估值快照/历史（用于趋势）

- 目的：支撑“净资产趋势”“历史对比”等趋势能力。
- 前端依赖的数据形态建议：
  - **快照（Snapshot）**：按天记录总资产/总负债/净资产，以及各类别汇总金额
  - **变更记录（History）**：记录资产项层面的估值变更（用于解释变化原因）
- 前端展示建议：
  - 总览趋势图默认读取快照数据（7/30/90 天）
  - 当日数据可由“当前汇总”覆盖最新值（避免当日快照尚未生成导致不够实时）
- 快照生成策略（供后端实现参考，但前端需约定）：
  - **按天快照**：每天生成 1 条（建议每日首次进入资产系统时生成；或每日固定时间生成）
  - 前端无需自行生成快照，只负责触发刷新与展示

## 6. 分类二级页（`/system/asset/categories/:category`）

- 顶部：分类名称 + 总金额 + 占比
- 列表：该分类下资产项
  - 字段：名称、金额、更新时间
  - 操作：编辑、删除（删除需二次确认）
- 空态：提示“添加第一笔资产”（引导使用页面内表单）
- 新增资产：沿用与主页面一致的字段结构

## 7. TanStack Query 约定（建议）

- 汇总：`['asset', 'summary']`
- 分类列表：`['asset', 'categories']`
- 分类详情：`['asset', 'category', categoryId]`
- 分类资产项：`['asset', 'items', categoryId]`
- 变更后刷新：
  - 新增/编辑/删除资产项 → `invalidateQueries(['asset', 'summary'])`
  - 同步刷新该分类详情/列表

## 8. MVP 范围（V1）

- 主页面：总资产大卡 + 净资产趋势 + 新增资产常驻表单
- 分类卡片：展示金额/占比/条目数，支持新增分类
- 分类二级页：资产项列表 + 新增/编辑/删除
- 快照：按天快照用于趋势
- 不包含：报表、设置、估值记录独立入口、AI 入口
