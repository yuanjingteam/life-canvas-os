# 营养师 Agent (Agentic RAG Pro) 行为驱动开发 (BDD) 文档

## 1. 引言
本文档描述了营养师 Agent 的核心业务行为，采用 Gherkin 语法（Given-When-Then）定义用户场景。本 BDD 文档旨在确保开发、测试与产品团队对系统行为达成共识，并作为自动化验收测试的依据。

## 2. 参与角色
- **用户 (User)**：寻求营养建议、记录体征或查询食物热量的最终使用者。
- **营养师 Agent (Agent)**：具备 RAG 检索、数学计算与画像分析能力的智能代理。
- **隐私网关 (Privacy Gateway)**：负责数据脱敏与本地/云端路由的底层安全模块。

---

## 3. 核心功能场景

### 场景一：精准营养数值计算 (Math Tool Tooling)
**需求背景**：消除大模型在计算 BMR, TDEE 等数值时的幻觉，确保建议基于科学公式。

**Feature: 精准计算 TDEE (总每日能量消耗)**
  **Scenario: 用户根据更新的体重请求计算 TDEE**
    **Given** 用户已登录且画像中包含身高 175cm、年龄 30岁、男性、活动系数为中度
    **And** 用户最近一次记录的体重为 80kg
    **When** 用户询问 "我的 TDEE 现在是多少？"
    **Then** Agent 必须调用 Python Math Tool 执行 Mifflin-St Jeor 公式
    **And** Agent 应当返回计算出的精确数值（如 2650 kcal）
    **And** 回答中必须明确标注 "经由数学工具计算得出"

---

### 场景二：基于知识库的营养建议 (Advanced RAG)
**需求背景**：提供符合《中国居民膳食指南》的专业、严谨的原则性建议。

**Feature: 针对特殊体质提供避障建议**
  **Scenario: 贫血用户咨询晚餐建议**
    **Given** 用户画像中的健康状况标记为 "轻度贫血"
    **And** 营养学向量知识库包含关于 "补血食材" 的权威指南
    **When** 用户询问 "我晚餐吃点什么能改善体质？"
    **Then** Agent 应当执行 RAG 检索召回补铁食材（如鸭血、瘦肉、菠菜）的段落
    **And** Agent 应当推荐富含维生素 C 的食物以促进铁吸收
    **And** Agent 必须在结尾附加标准医疗免责声明

---

### 场景三：时序动态策略调整 (Temporal Profile)
**需求背景**：系统能感知用户体征变化趋势并自动切换干预强度。

**Feature: 检测到体重波动并调整热量缺口**
  **Scenario: 连续一周减重过快触发健康警告**
    **Given** 用户处于 "减脂模式"
    **And** HealthLogs 显示过去 7 天体重下降超过 2kg（超过安全阈值）
    **When** 用户询问 "明天的饮食计划是什么？"
    **Then** Agent 应当通过趋势分析识别到 "减重过快" 的风险
    **And** Agent 应当自动将热量缺口从 500kcal 调整为 200kcal（恢复性模式）
    **And** Agent 应当建议用户增加碳水化合物摄入并提醒注意疲劳感

---

### 场景四：事实性食物查询 (SQL Router)
**需求背景**：对于结构化数据，绕过 RAG 直接查询数据库以保证数据权威性。

**Feature: 查询特定食物热量**
  **Scenario: 用户查询常见食物的蛋白质含量**
    **Given** 系统连接了标准的结构化食物成分数据库 (SQL)
    **When** 用户询问 "100克鸡胸肉有多少蛋白质？"
    **Then** Router 应当将请求路由至 SQL Connector 检索精确值
    **And** Agent 应当返回 "23.1克"（来源于数据库原始记录）
    **And** Agent 不应当在此时检索长文本 RAG 文献

---

### 场景五：隐私保护与脱敏 (Privacy Gateway)
**需求背景**：保护用户健康隐私，确保敏感信息不出本地。

**Feature: 敏感健康数据脱敏处理**
  **Scenario: 在调用公网大模型前处理隐私信息**
    **Given** 用户正在咨询关于特定疾病（如乙肝）的饮食禁忌
    **And** 用户的个人真实姓名记录在画像中
    **When** Agent 准备将查询请求发送至云端 LLM
    **Then** 隐私网关必须拦截请求
    **And** 隐私网关必须将姓名替换为占位符（如 [USER_ID]）
    **And** 隐私网关必须将特定疾病名称转换为通用的医学分类标签

---

## 4. 验收标准 (Acceptance Criteria)
1. **数值一致性**：所有数学计算结果必须与纯 Python 脚本执行结果误差小于 0.1%。
2. **检索准确率**：RAG 召回的知识片段必须与用户问题的相关性得分（Reranker Score）大于 0.75。
3. **时序响应**：当 HealthLogs 出现异常波动时，Agent 必须在首轮对话中主动提示风险。
4. **零泄露**：发送至公网的消息体中不得包含任何明文 PII 信息。
